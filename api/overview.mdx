---
title: "REST API overview"
description: "PhoneInfoga ships a REST API you can use for programmatic phone number scanning without the web client."
---

PhoneInfoga ships a REST API that lets you parse, validate, and scan phone numbers programmatically. The API is served by the same `phoneinfoga serve` command that powers the web client.

All endpoints accept and return JSON. No authentication is required — the server is designed for local or trusted-network use.

## Starting the server

```bash
phoneinfoga serve
```

The server starts on port `5000` by default. To use a different port, pass the `-p` flag:

```bash
phoneinfoga serve -p 8080
```

To serve the REST API without the web client:

```bash
phoneinfoga serve --no-client
```

To run the server in Docker:

```bash
docker run --rm -it -p 5000:5000 sundowndev/phoneinfoga serve
```

## Base URL

```
http://localhost:5000
```

All API paths are relative to this base URL. If you change the port with `-p`, update the base URL accordingly.

## Response format

Every response is a JSON object. On success, the response body contains the endpoint-specific fields documented below. On failure, the response body is:

```json
{"error": "message"}
```

The HTTP status code indicates the outcome:

| Status | Meaning |
|--------|---------|
| `200` | Success |
| `400` | Invalid input (e.g. malformed phone number) |
| `404` | Resource not found (e.g. unknown scanner name) |
| `500` | Internal server error |

## Available endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/v2/numbers` | Parse and validate a phone number |
| `GET` | `/api/v2/scanners` | List all available scanners |
| `POST` | `/api/v2/scanners/{scanner}/dryrun` | Check if a scanner would run for a number |
| `POST` | `/api/v2/scanners/{scanner}/run` | Run a specific scanner against a number |

See [Numbers](/api/numbers) and [Scanners](/api/scanners) for full endpoint documentation.

## Scanner options

Scanner credentials can be passed per-request via the `options` field in the request body. Option keys are named exactly like the corresponding environment variables (e.g., `NUMVERIFY_API_KEY`, `GOOGLECSE_CX`).

<Note>
  Per-request options override environment variables for that request only. They do not affect other concurrent requests or the server's global configuration.
</Note>
