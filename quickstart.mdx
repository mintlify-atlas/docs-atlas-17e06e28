---
title: "Quickstart"
description: "Run your first phone number scan in under 2 minutes."
---

This guide walks you through installing PhoneInfoga, running your first scan, and optionally launching the web interface.

<Steps>
  <Step title="Install PhoneInfoga">
    Download and install the binary on UNIX systems with a single command:

    ```bash
    bash <( curl -sSL https://raw.githubusercontent.com/sundowndev/phoneinfoga/master/support/scripts/install )
    sudo install ./phoneinfoga /usr/local/bin/phoneinfoga
    ```

    For other installation methods — Homebrew, Docker, or building from source — see the [Installation](/installation) page.
  </Step>
  <Step title="Run a scan">
    Use the `scan` command with the `-n` flag to scan a phone number. The number can be in E.164 or international format — special characters such as `( ) - +` are handled automatically.

    ```bash
    phoneinfoga scan -n "+1 (555) 444-1212"
    phoneinfoga scan -n "+33 06 79368229"
    ```

    <Note>
      The country code is required. If you are unsure of the country code to use, find it at [countrycode.org](https://www.countrycode.org/).
    </Note>

    By default, only scanners that are already configured will run. The local scanner runs without any configuration and provides basic number validation, country, carrier, and line type. To get deeper results from external data sources, configure additional scanners — see [Scanners overview](/scanners/overview).
  </Step>
  <Step title="Launch the web server (optional)">
    PhoneInfoga includes a web UI built with Vue.js and a REST API. Start the server on the default port 5000:

    ```bash
    phoneinfoga serve
    ```

    To use a different port:

    ```bash
    phoneinfoga serve -p 8080
    ```

    Open your browser at `http://localhost:5000` (or the port you specified) to use the graphical interface.

    To run the REST API without the web client:

    ```bash
    phoneinfoga serve --no-client
    ```

    For Docker equivalents and full configuration options, see [Web server](/usage/web-server).
  </Step>
</Steps>
